# 섹션 1: 프로젝트와 프레임워크의 이해
## 프로젝트란?
* 정의 : 하나의 목표를 세우고, 이를 달성하기 위한 모든 과정
* 업무의 묶음(Process) : 일정 기간 내에 특정 목적을 달성하기 위한 유기적인 작업 단위
* 범위 : 하나의 완성된 프로그램부터 작은 기능 구현, 에러 해결(Debugging), 신규 기능 추가 등 모든 개발 활동을 포함
 
## 프레임워크 (Framework)
* 의미 : 프레임(Frame, 틀) + 워크(Work, 일) = **"정해진 틀 안에서 일하는 것"**
* 장점 : 개발자가 기초적인 구조를 직접 설계할 필요 없이, 이미 마련된 틀 안에서 필요한 기능을 사용하므로 개발 속도와 효율이 비약적으로 향상됨
* 핵심 : 다양한 **라이브러리**가 미리 준비되어 있어, 개발자는 **비즈니스 로직(Logic) 구현에만 집중**할 수 있음

## 라이브러리 (Library)
* 선배 개발자들이 미리 만들어 둔 효율적인 코드 꾸러미를 빌려 쓰는 것 의미

# 색션 2: 스프링 프레임워크와 스프링 부트
## 스프링 프레임워크 (Spring Framework)
* 등장 배경 : 기존 자바 개발의 불편함을 해결하기 위해 등장
* 과거의 불편함 :
    * 많은 기능을 사용하기 위해 개발자가 복잡한 설정을 일일이 직접 해야 했음
    * 웹 환경에 특화된 설정을 구성하기 매우 까다로움
  
## 스프링 부트
* 특징 : 더욱 **개발자 지향적**으로 진화한 형태
* 주요 장점 :
    1. 자동화 : 과거 스프링에서 직접 해야 했던 수많은 설정이 자동화되었음
    2. 웹 특화 : 웹 애플리케이션 제작에 최적화되어 있음 (물론 웹 외의 용도로도 사용 가능)
